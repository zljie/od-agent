# johnosn SDD Constitution

## Core Principles

### I. Result-Oriented (结果导向)

一旦通过工具调用获得关键信息（如报错代码、配对码、成功回执），**必须立即停止后续探测**并向用户汇报。严禁进行无意义的后台健康检查。

**具体要求**:
- 获得明确结果（成功/失败/错误码）后立即停止轮询
- 优先报告结果，再处理后续分析
- 不进行冗余的状态确认

### II. No Busy-Wait (禁止无效轮询)

在排查连接或等待异步结果时，**禁止使用 sleep 超过 5 秒的循环探测**。如果 15 秒内无日志更新，请立即停止并请求人工介入。

**具体要求**:
- 最大单次等待: 5 秒
- 最大总等待时间: 15 秒
- 超时后必须请求人工介入，禁止继续自动重试

### III. Read Before Act (调研先行原则)

在执行安装、修改配置或重启服务前，**必须先查阅本地文件**（ls / cat / grep）和历史日志。禁止在未确认环境的情况下盲目执行写操作或重复重启。

**具体要求**:
- 安装前: 检查现有依赖、版本、配置
- 修改配置前: 读取当前配置、备份原值
- 重启服务前: 检查日志、确认问题根因
- 禁止: 盲目重启、重复执行相同操作

### IV. Token Saving (上下文精简)

调用 exec 或 read 读取日志时，若输出超过 20 行，**必须自行筛选核心关键词行**。严禁将数百行原始日志全量传回上下文。

**具体要求**:
- 超过 20 行: 使用 grep/awk 筛选核心关键词
- 保留: ERROR, FATAL, exception, timeout, success, failed
- 丢弃: DEBUG 详情、重复堆栈、中间过程日志

### V. Skip COT (禁止过度思考)

对于"检查状态"、"重启网关"、"读取文件"等**标准运维操作**，跳过复杂的思维链（Thinking Process）分析，直接执行并给出结果。

**具体要求**:
- 运维操作: 直接执行，无需先分析
- 复杂问题: 需要分析，但限制思考步骤
- 代码问题: 先读代码，再分析

## Development Workflow

### SDD Process

项目采用 Spec-Driven Development 工作流:

```
1. /speckit.constitution - 项目原则 (本文件)
2. /speckit.specify - 定义需求和用户故事
3. /speckit.plan - 技术方案规划
4. /speckit.tasks - 任务拆分
5. /speckit.implement - 执行实现
```

### Iteration Board

使用 `plan.md` 作为轻量迭代看板:
- 每次开始新任务新增卡片
- 实现完成并通过验收后，移动到 Done 并勾选验收标准
- 保持 `plan.md` 与项目架构一致

## Governance

- 本规范优先于其他实践准则
- 复杂决策需要充分论证
- 运维操作必须遵循以上五条核心原则

## 8. Harness 复用与模仿规范 (Harness-Driven Reuse & Imitation)

### 8.1 找相似实现 (Search for Similar Implementations)

- 在开始设计或编码前，必须先在代码库中搜索与当前需求功能相似的现有实现（`grep`、语义搜索、或利用 Cursor Codebase Search）。
- 将搜索到的相似代码路径作为技术方案的基准参考，在 `plan.md` 中显式引用。

### 8.2 复用优先 (Reuse First)

- 对于已存在的组件、接口、数据结构、工具函数，只要满足需求，必须直接复用，不得重写。
- 若现有实现与需求有微小偏差，优先考虑通过参数化或配置扩展实现复用，而非新建平行组件。

### 8.3 模仿者复制 (Copy & Adapt, Don't Write Fresh)

- 当必须新建文件或模块时，必须从最相似的现有代码文件中复制一份，然后在其基础上修改。
- 严格保持与参考代码一致的命名风格、错误处理模式、日志格式和目录结构。
- 禁止在无相似参考的情况下凭空设计新架构，除非明确批准。

## 三大纪律总结

本宪章整合了三大纪律，所有通过 Cursor + Spec Kit 的新项目，只需复制 `.specify/memory/constitution.md` 和 `.cursor/rules/` 文件夹，即可自动继承这套完整的行为规范：

1. **高效率运维准则** (结果导向、禁轮询、读后行、省 Token、跳过 COT)
2. **Harness 复用规范** (找相似、复用优先、模仿复制)
